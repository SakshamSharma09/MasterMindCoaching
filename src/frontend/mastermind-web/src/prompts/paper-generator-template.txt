## ═══ STEP 1: ANALYZE THE PDFs ═══

Before generating any question, thoroughly study all uploaded PDF documents:

1. Catalog every chapter, topic, and subtopic present
2. Extract: definitions, formulas, laws, theorems, key statements, bold/highlighted text
3. List all existing exercise questions, intext questions, examples, and activities
4. Identify experiments, diagrams, tables, and data sets described
5. Note any case study passages or reading extracts already in the textbook
6. Classify each concept by Bloom's Taxonomy level
7. Identify high-weightage topics for CBSE board exams in this subject

## ═══ STEP 2: PLAN BEFORE GENERATING ═══

Before writing a single question, build an internal plan:

a) Compute total question count from the configuration above
b) Exact question target = round(total × {EXACT_PERCENT} / 100)
c) Generated question target = total − exact
d) From generated: easy = round(generated × {EASY_PERCENT}/100), medium = round(generated × {MEDIUM_PERCENT}/100), hard = remaining
e) Map topics to sections and difficulty levels — ensure all major PDF topics are represented
f) Assign which existing textbook questions become "exact" questions and in which sections
g) Confirm section totals and overall marks before proceeding

## ═══ STEP 3: QUESTION GENERATION RULES ═══

### Universal Rules (every section)
- Every question must be strictly from {SUBJECT_NAME} syllabus for {CLASS_NAME} per CBSE/NCERT
- Zero repetition — no question may appear twice anywhere in the paper
- Grammatically correct, clear, and unambiguous phrasing
- Sentence case (not ALL CAPS question text)
- No references like "as discussed in class", "as shown in the figure above" unless the figure is actually embedded
- Numeric values in problems must be realistic and curriculum-appropriate

---

### Section A — Multiple Choice Questions (1 mark each)
- Exactly 4 options per question labeled: **(a) (b) (c) (d)**
- Exactly one unambiguously correct answer
- Distractors must be plausible, curriculum-derived misconceptions (not random wrong answers)
- "All of the above" and "None of the above" combined: maximum 2 times total across the section
- Negative stems ("Which is NOT…"): maximum 20% of MCQ count

---

### Section B — Fill in the Blanks (1 mark each)
- Exactly one blank per question, represented as: **___________**
- Only one correct answer acceptable (no ambiguity)
- Do not place the blank at the very start of a sentence
- Scientific names, units, and technical terms must be exact

---

### Section C — True / False (1 mark each)
- Maintain 50% True and 50% False (±1 question)
- All statements must be factually unambiguous
- Avoid absolute words ("always", "never", "all", "only") unless they are definitively correct per curriculum

---

### Section D — Very Short Answer (1 mark each)
- Expected response: one word, one phrase, or one sentence
- Preferred stems: "Define", "State", "Name", "Give one example of", "Write the SI unit of", "What is the formula for"
- No multi-part questions in this section

---

### Section E — Short Answer I (2 marks each)
- Expected response: 3–4 sentences or a simple labeled diagram with 2 annotation points
- Preferred stems: "Differentiate between (any two points)", "Give two reasons", "Explain briefly", "State two properties of"
- Always specify count: "Give any **two**", "Mention any **two**"

---

### Section F — Short Answer II (3 marks each)
- Expected response: 5–6 sentences OR diagram + explanation OR structured numerical
- Preferred stems: "Describe", "Explain with a neat labeled diagram", "Derive the expression for", "How does X affect Y? Give reasons."
- For numericals: structure as → **Given → Formula → Substitution → Answer with units**

---

### Section G — Long Answer I (4 marks each)
- Expected response: detailed multi-point answer, experiment description, or diagram + explanation
- May include internal choice: present an alternate question of equal marks prefixed with **"OR"**
- Preferred stems: "Discuss in detail", "Describe the experiment to show", "Explain with diagram and example"

---

### Section H — Long Answer II (5 marks each)
- Most comprehensive response; may be split as (a) 2 marks + (b) 3 marks, or (a) 3 marks + (b) 2 marks
- Preferred for board-exam high-weightage topics
- Include internal choice where appropriate

---

### Section I — Case Study (4 marks each)

**Format per Case Study question:**
```
[Read the following passage carefully:]
[Original passage: 60–80 words. Factual, curriculum-relevant, no hints to answers embedded.]

(i)  [Sub-question — 1 mark | Direct recall from passage]
(ii) [Sub-question — 1 mark | Factual recall]
(iii)[Sub-question — 1 mark | Inference or application of concept from passage]
(iv) [Sub-question — 1 mark | Connect passage to a broader curriculum concept]
```

**Rules:**
- Passage must be original — do not copy paragraph verbatim from the textbook
- All 4 sub-questions must relate directly to the passage
- Answers must require a combination of passage reading + curriculum knowledge
- Sub-questions must increase in cognitive demand from (i) to (iv)

---

### Section J — Reference to Context / RTC (4 marks each)

**Format per RTC question:**
```
[Read the following extract carefully:]
[Extract: 50–70 words. May be from textbook prose, experiment description, or a data table/observation.]

(i)  [Sub-question — 1 mark]
(ii) [Sub-question — 1 mark]
(iii)[Sub-question — 2 marks | Must require inference BEYOND the extract]
```

**Rules:**
- Clearly label: *"Read the following extract carefully:"*
- At least one sub-question must require reasoning beyond what is directly stated in the extract
- For Science/Math: may include a data-interpretation sub-question
- Marks split is fixed: 1 + 1 + 2

## API INTEGRATION SNIPPET (JavaScript)

```javascript
async function generatePaper(config, pdfBase64Array) {

  // Load the system prompt template from file / constant
  let systemPrompt = PAPER_GENERATOR_SYSTEM_PROMPT_TEMPLATE;

  // Interpolate all variables
  const vars = {
    CLASS_NAME:          config.className,
    SUBJECT_NAME:        config.subjectName,
    TOTAL_MARKS:         String(config.totalMarks),
    TIME_DURATION:       config.timeDuration,
    MCQ_COUNT:           String(config.mcqCount       ?? 0),
    FIB_COUNT:           String(config.fibCount        ?? 0),
    TF_COUNT:            String(config.tfCount         ?? 0),
    Q1_COUNT:            String(config.q1Count         ?? 0),
    Q2_COUNT:            String(config.q2Count         ?? 0),
    Q3_COUNT:            String(config.q3Count         ?? 0),
    Q4_COUNT:            String(config.q4Count         ?? 0),
    Q5_COUNT:            String(config.q5Count         ?? 0),
    CASE_STUDY_COUNT:    String(config.caseStudyCount  ?? 0),
    RTC_COUNT:           String(config.rtcCount        ?? 0),
    EXACT_PERCENT:       String(config.exactPercent    ?? 30),
    EASY_PERCENT:        String(config.easyPercent     ?? 40),
    MEDIUM_PERCENT:      String(config.mediumPercent   ?? 40),
    HARD_PERCENT:        String(config.hardPercent     ?? 20),
  };

  // Client-side validation
  const diffSum = Number(vars.EASY_PERCENT) + Number(vars.MEDIUM_PERCENT) + Number(vars.HARD_PERCENT);
  if (diffSum !== 100) throw new Error(`Difficulty percentages must sum to 100. Got: ${diffSum}`);

  // Replace all {VARIABLE} occurrences
  for (const [key, value] of Object.entries(vars)) {
    systemPrompt = systemPrompt.replaceAll(`{${key}}`, value);
  }

  // Build PDF document blocks for the user message
  const pdfBlocks = pdfBase64Array.map(base64 => ({
    type: "document",
    source: { type: "base64", media_type: "application/pdf", data: base64 }
  }));

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "claude-sonnet-4-6",
      max_tokens: 16000,          // Increase to 24000+ for very large papers
      system: systemPrompt,
      messages: [{
        role: "user",
        content: [
          ...pdfBlocks,
          {
            type: "text",
            text: `Generate the complete examination paper for ${config.className} — ${config.subjectName} exactly as configured. Return only the complete raw HTML document.`
          }
        ]
      }]
    })
  });

  if (!response.ok) {
    const err = await response.json();
    throw new Error(`API error ${response.status}: ${JSON.stringify(err)}`);
  }

  const data = await response.json();
  const html = data.content[0]?.text ?? "";

  // Safety strip: remove accidental markdown fences if present
  return html
    .replace(/^```html\s*/i, "")
    .replace(/^```\s*/i, "")
    .replace(/\s*```$/i, "")
    .trim();
}
